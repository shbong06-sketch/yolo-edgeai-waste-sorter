# 프로젝트 제안: YOLO 기반 에지 AI 쓰레기 자동 분류 시스템

## 1. 프로젝트 개요

### 프로젝트 명

- YOLO 기반 에지 AI 쓰레기 자동 분류 시스템

#### 프로젝트 목적

- 카메라 피드를 통해 유입되는 폐기물을 **[금속 캔, 페트병, 스티로폼]** 3가지 클래스로 실시간 객체 탐지(Object Detection)하고 분류하는 경량화된 AI 모델 구축 및 SO-ARM 101 로봇 팔 연동을 통한 자동 분류 매니퓰레이터 시스템 구현

## 2. 문제 정의

- 스마트 팩토리 및 자동화 공정에서 재활용품 분리수거는 여전히 수작업 의존도가 높고 비용이 많이 드는 영역이다.
- 특히 작업자의 안전 문제와 구인난을 해결하기 위해, 비전 AI와 로봇 공학을 결합한 고속 자동 선별 시스템의 도입이 시급하다.

## 3. 목표

### MVP(반드시 구현)

#### 데이터셋 구축

- AIHub 데이터를 활용한 [금속 캔, 페트병, 스티로폼] 3종 맞춤형 데이터셋 구축
    - 재활용품 분류 및 선별 데이터
        - [재활용품 분류 및 선별 데이터](https://www.aihub.or.kr/aihubdata/data/view.do?pageIndex=1&currMenu=115&topMenu=100&srchOptnCnd=OPTNCND001&searchKeyword=%EB%B6%84%EB%A6%AC%EC%88%98%EA%B1%B0&srchDetailCnd=DETAILCND001&srchOrder=ORDER001&srchPagePer=20&aihubDataSe=data&dataSetSn=71362)

#### AI

- 실시간 추론을 위한 YOLO 객체 탐지 모델 학습 및 성능 확보

### Stretch Goal(추가 목표)

#### Simulation

- 컨베이어 벨트 및 로봇 통합
    - SO-ARM 101 로봇 팔과 소형 가상/실물 컨베이어 벨트를 연동합니다.
- 비전-행동(Vision-to-Action) 구현
    - 카메라가 벨트 위로 지나가는 쓰레기를 탐지하면, 해당 픽셀 좌표를 로봇 팔의 3D 공간 좌표계로 변환(Coordinate Transformation)
    - 변환된 타겟 좌표를 기반으로 로봇 마니퓰레이터가 그리핑 액션을 취해, 쓰레기를 각각 지정된 수거함 위치로 분류하는 완전 자동화 공정 구현 (시뮬레이션 및 실물)

## 4. 시스템 구조/흐름도

```python
[ 데이터/학습 파이프라인 (Offline) ]

		AIHub 데이터 수집
                │
                ▼
		데이터 전처리/라벨링
                │
                ▼
			YOLO 모델 학습
                │
                ▼
    경량화 모델 최적화 (.pt/.onnx)
                │
                ▼
[ 실시간 비전-행동 제어 루프 (Online) ]

		컨베이어 벨트 구동
                │
                ▼
		비전 카메라 스트리밍
                │
                ▼
		이미지 프레임 입력
                │
                ▼
        YOLO 객체 탐지 ------------- [ AI 추론 엔진 ]
                │ (클래스, 2D 좌표)
                ▼
	3D 공간 좌표 변환 (Homography)
                │
                ▼
		ROS2 로봇 팔 제어
                │
                ▼
        수거함 배치 완료
```

하드웨어 및 소프트웨어 통합 구조도

| **레이어 (Layer)** | **구성 요소** | **데이터 흐름 및 역할** |
| --- | --- | --- |
| **1. 입력단 (Input)** | 컨베이어 벨트, RGB 비전 카메라 | 벨트 위로 이동하는 폐기물 영상을 실시간(FPS)으로 캡처하여 스트리밍 데이터로 전송 |
| **2. 인지단 (Perception)** | PC / 에지 디바이스 (YOLO Engine) | 입력된 프레임에서 [금속 캔, 페트병, 스티로폼]을 탐지하고, 바운딩 박스의 중심점 픽셀 좌표 (x, y) 추출 |
| **3. 제어단 (Control)** | 좌표 변환 모듈, ROS2 프레임워크 | 카메라 2D 픽셀 좌표 (x, y)를 로봇 작업 공간의 3D 물리 좌표 (X, Y, Z)로 변환 후, 로봇 역기하학(Inverse Kinematics) 기반 궤적 생성 |
| **4. 구동단 (Action)** | SO-ARM 101 마니퓰레이터 | 생성된 궤적을 따라 그리퍼를 이동하여 타겟을 집고(Gripping), 지정된 쓰레기 수거함 위치로 이동 및 분류 |

## 6. 역할 분배

| **담당 역할** | **핵심 수행 태스크** |
| --- | --- |
| **AI 모델 개발 (Model)** | 데이터 전처리/증강 가이드라인 설계, YOLO 모델 파인 튜닝 및 성능 평가 지표(mAP, IoU) 분석 |
| **하드웨어 및 인프라 (HW/Cloud)** | 비전 카메라 센서 셋업, ROS2 프레임워크 기반 SO-ARM 101 액추에이터 제어 및 공간 좌표 변환 궤적 설계 |
| **풀스택 및 인터페이스 (Full-Stack)** | 실시간 비전 추론 스트리밍 대시보드 웹 개발, 상태 피드백 제어 인터페이스 및 데이터 로그 모니터링 구축 |

## 7. 일정 및 우선순위

- 1주차: 개발 환경 세팅 및 데이터셋/베이스라인 구축
    - GitHub 레포지토리 및 RoboFlow 워크 스페이스 생성
    - 커스텀 데이터셋 구축: AIHub 데이터 가공(금속캔, 페트병, 스티로폼 3종 선별)
        - 데이터 수량 구체화: 팀원 4명이 클래스 별 각 125장씩 수집/라벨링 → 클래스별 500장의 이미지 데이터셋 확보(총 1500장 크기의 데이터셋)
        - 맡은 번호: 승현(0~124) 성현(1000~1124) 범진(2000~2124) 세교(3000~3124) 데이터 생성
    - 샘플 데이터를 활용한 YOLO 객체 탐지 기본 파이프라인 구성 (~7/6 1차 공유)
- 2주차: 모델 파인튜닝 및 비전-행동 통합 연동
    - AI: 데이터 증강 및 YOLO 모델 본격 학습, 성능 평가(mAP, IoU) 및 모델 경량화
    - HW: 훈련된 모델을 바탕으로 로봇과 연결
        - 카메라-로봇 간 좌표 변환(Homography) 알고리즘 구현, ROS2 기반 로봇 제어 궤적 설계 (더미 데이터로 선행 테스트)
    - 통합: 주말 중 학습된 모델을 ROS2 시스템에 얹어 실물/시뮬레이션 로봇과 연동 시작
- 3주차: 시스템 고도화, 테스트 및 최종 산출물 정리
    - 분류 정확도 및 로봇 그리핑 성공률 테스트/디버깅
    - 최종 결과물 산출: 최종 데모 영상 촬영 및 프로젝트 보고서 산출
