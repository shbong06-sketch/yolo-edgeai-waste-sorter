# 프로젝트 제안: YOLO 기반 에지 AI 쓰레기 자동 분류 시스템

## 1. 프로젝트 개요

### 프로젝트 명

- YOLO 기반 에지 AI 쓰레기 자동 분류 시스템

#### 프로젝트 목적

- 카메라 피드를 통해 유입되는 폐기물을 **[금속 캔, 페트병, 스티로폼]** 3가지 클래스로 실시간 객체 탐지(Object Detection)하고 분류하는 경량화된 AI 모델 구축 및 SO-ARM 101 로봇 팔 연동을 통한 자동 분류 매니퓰레이터 시스템 구현

## 2. 문제 정의

- 스마트 팩토리 및 자동화 공정에서 재활용품 분리수거는 여전히 수작업 의존도가 높고 비용이 많이 드는 영역이다.
- 특히 작업자의 안전 문제와 구인난을 해결하기 위해, 비전 AI와 로봇 공학을 결합한 고속 자동 선별 시스템의 도입이 시급하다.

#### Simulation

- 컨베이어 벨트 및 로봇 통합

- 비전-행동(Vision-to-Action) 구현
    - 카메라가 벨트 위로 지나가는 쓰레기를 탐지하면, 해당 픽셀 좌표를 로봇 팔의 3D 공간 좌표계로 변환(Coordinate Transformation)
    - 변환된 타겟 좌표를 기반으로 로봇 마니퓰레이터가 그리핑 액션을 취해, 쓰레기를 각각 지정된 수거함 위치로 분류하는 완전 자동화 공정 구현 (시뮬레이션 및 실물)

## 4. 시스템 구조/흐름도

```python
[ 데이터/학습 파이프라인 (Offline) ]

	  	AIHub 데이터 수집
	  					│
		  				▼
		  데이터 전처리/라벨링
			  			│
			  			▼
			  YOLO 모델 학습
			  			│
			  			▼
  경량화 모델 최적화 (.pt/.onnx)
		  				│
			  			▼
[ Isaac Sim 가상 검증 파이프라인 (Virtual Sandbox) ]

    Isaac Sim 가상 컨베이어 & 로봇 셋업
               │
               ▼
	가상 카메라 스트리밍 및 YOLO 추론 연동
               │
               ▼
 3D 좌표 변환 및 ROS2 가상 로봇 제어
               │
               ▼
						
[ 실물 비전-행동 제어 루프 (Physical Online) ]

		실물 컨베이어 벨트 구동
               │
               ▼
  		비전 카메라 스트리밍
               │
               ▼
        YOLO 객체 탐지 ------------- [ AI 추론 엔진 ]
               │ (클래스, 2D 좌표)
               ▼
	3D 공간 좌표 변환 (Homography)
               │
               ▼
 		ROS2 실물 로봇 팔 제어
               │
               ▼
        수거함 배치 완료
```

하드웨어 및 소프트웨어 통합 구조도

| **레이어 (Layer)** | **구성 요소** | **데이터 흐름 및 역할** |
| --- | --- | --- |
| **1. 입력단 (Input)** | 컨베이어 벨트, RGB 비전 카메라 | 벨트 위로 이동하는 폐기물 영상을 실시간(FPS)으로 캡처하여 스트리밍 데이터로 전송 |
| **2. 인지단 (Perception)** | PC / 에지 디바이스 (YOLO Engine) | 입력된 프레임에서 [금속 캔, 페트병, 스티로폼]을 탐지하고, 바운딩 박스의 중심점 픽셀 좌표 (x, y) 추출 |
| **3. 검증단 (Simulation)** | Isaac Sim 가상 환경 | 실물 가동 전, 가상 컨베이어 벨트와 로봇 팔을 매핑하여 비전-행동 알고리즘 사전 검증 및 디버깅 |
| **4. 제어단 (Control)** | 좌표 변환 모듈, ROS2 프레임워크 | 카메라 2D 픽셀 좌표를 로봇 작업 공간의 3D 물리 좌표로 변환 후, 로봇 역기하학(Inverse Kinematics) 기반 궤적 생성 |
| **5. 구동단 (Action)** | SO-ARM 101 마니퓰레이터 | 생성된 궤적을 따라 그리퍼를 이동하여 타겟을 집고(Gripping), 지정된 쓰레기 수거함 위치로 이동 및 분류 |

## 9. 성공 기준

- 성공 기준 고려 사항(Feat. 제미나이) 이야기 나눌 때 참고용
    
    에지 AI 및 로봇 제어 학술 자료와 최신 논문(2025~2026년 발표 기준)들을 바탕으로 분석해 보면, 학계와 산업계에서 "비전 기반 로봇 선별 시스템(Vision-Guided Robotic Sorting System)"의 목표치를 설정할 때 공통으로 사용하는 표준 가이드라인이 있습니다.
    
    이번 프로젝트의 환경(3주 개발 기간, 500장의 데이터셋 규모, 소형 로봇 팔)을 고려하여 논문 데이터를 기반으로 가장 이상적인 목표치 설정법을 제안해 드립니다.
    
    ## 📑 학술 자료를 통해 본 3대 핵심 목표 설정법
    
    ### 1. AI 모델 정확도: mAP@0.5 (Mean Average Precision)
    
    학습 데이터셋의 규모와 환경 제어 여부에 따라 목표치를 현실적으로 조정해야 합니다.
    
    - **학술적 배경:** 최근 연구들에서 대규모 데이터셋(10,000장 이상)과 고성능 GPU 기반의 YOLO 모델은 mAP 90% 이상을 쉽게 달성합니다. 하지만 쓰레기 분류 관련 논문(*A Deep Learning Based Real-Time Model for Automatic Waste Segregation*)에 따르면, **작거나 정제된 커스텀 데이터셋을 활용해 다중 클래스를 분류할 때의 실질적인 mAP는 대략 72% ~ 81% 선**을 기록합니다.
    - **추천 목표 설정:** 데이터셋이 클래스당 약 150~170장(총 500장)으로 시작하는 초기 단계이고 가메라 탑뷰(Top-down) 환경이 일정하므로, mAP@0.5 기준 75% ~ 80%를 현실적인 1차 Target으로 잡는 것이 학술적으로 방어하기 좋습니다.
    
    ### 2. 추론 속도: Latency 및 FPS (Frames Per Second)
    
    에지 AI(Edge AI) 타이틀을 달기 위해서는 모델의 정확도만큼 '지연 시간'이 중요합니다.
    
    - **학술적 배경:** 컨베이어 벨트 위에서 움직이는 물체를 놓치지 않고 그리핑 지점을 계산하려면 최소 **20~30 FPS** 이상의 속도가 보장되어야 합니다. 최신 YOLO 경량화 버전을 다룬 논문들(YOLOv8n, YOLOv11n 등)은 에지 디바이스 단에서 10ms~30ms 사이의 인퍼런스 타임(추론 지연 시간)을 표준 성공 기준으로 잡습니다.
    - **추천 목표 설정:** 프레임 수치인 `30 FPS 이상`도 좋지만, 하드웨어 제어와의 연동을 고려해 **"단일 프레임당 추론 지연 시간(Inference Latency) 35ms 이하 확보"** 형태로 명시하는 것이 훨씬 엔지니어링 관점에 부합합니다.
    
    ### 3. 로봇 구동 및 매니퓰레이션 성공률 (Pick-and-Place Success Rate)
    
    비전의 성능이 아무리 좋아도 물리적인 로봇 팔의 오차 때문에 성공률이 떨어지는 현상이 가장 빈번하게 발생합니다.
    
    - **학술적 배경:** 2026년 최신 논문인 *Vision-Guided Robotic System for Automated Waste Sorting* 에 따르면, YOLO 모델과 로봇 팔(6 DOF)을 통합 구동했을 때 체커보드를 활용한 핸드-아이 캘리브레이션(Hand-Eye Calibration)을 정밀하게 수행하면 포지셔닝 오차를 ±5.2mm까지 줄여 **약 92.5% ~ 95%의 그리핑 성공률**을 보인다고 보고되어 있습니다.
    - **추천 목표 설정:** 다만 사용하시는 SO-ARM 101은 소형 액추에이터 기반이고 컨베이어 동적 환경을 처음 마주하므로, 논문 수준인 90%보다는 조금 보수적으로 접근해 "종합 분류 성공률 80% 이상 (20회 시도 중 16회 이상 성공)"으로 잡는 것이 기획서 검토 시 신뢰감을 줍니다.
    
    ### 예시) 9. 성공 기준 (학술 자료 기반 튜닝)
